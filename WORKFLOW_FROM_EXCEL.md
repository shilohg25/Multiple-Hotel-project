# Workflow mapped from attached Excel samples

The app was designed around the repeated workflow found in the uploaded spreadsheets.

## Reservation tracker pattern

Source file: `TAGOSILANGAN NEW RESERVATIONS(1).xlsx`

Repeated columns mapped to the app:

| Spreadsheet column | App field |
|---|---|
| Reservation Method | `reservations.booking_source` |
| Room Type | `rooms.name` / `rooms.room_type_name` |
| With Breakfast? | `reservations.with_breakfast` |
| Guest Name/s | `guests.full_name` |
| Check-in Date | `reservations.check_in` |
| Check-out Date | `reservations.check_out` |
| # of Nights | generated by database as `reservations.nights` |
| # of Pax | `reservations.guest_count` |
| Posted Room Rate | `reservations.posted_room_rate` |
| Addtl Surcharge | `reservations.surcharge_label` |
| Surcharge Amount | `reservations.surcharge_amount` |
| Total Amount | `reservations.total_amount` |
| Downpayment | `reservations.downpayment_required` plus payment records |
| Status of Payment | `payments.status` and `reservations.status` |
| Outstanding Balance/s | computed in reservation detail page |
| Mode of Payment | `payments.method` |
| Reservation Confirmed By | audit trail / confirming staff |

## Collectibles / payment follow-up pattern

Source file: `COLLE(CTABLES) NAVARRO(1).xlsx`

The sheet tracks guest, company/source, room, total amount, partial payments, outstanding balance, and status. In the app this is handled through:

- `reservations.total_amount`
- `payments.amount`
- `payments.status`
- computed confirmed payment total
- computed balance
- reservation status rules: tentative → payment submitted → secured

## Daily calculation / front-desk sales pattern

Source file: `NAVARRO Format Calculation(1).xlsx`

The app includes `ledger_entries` and `cash_counts` tables for the daily calculation workflow. The current first version focuses the UI on reservations and payments; the schema is ready for a Daily Sales screen using these same tables.

## Added requirements implemented

- Multi-hotel architecture with owner-level hotel creation.
- Staff login via Supabase Auth.
- Online booking page per hotel: `/book/[hotelSlug]`.
- Payment proof is mandatory for online booking and payment recording.
- Booking remains tentative / payment review until payment confirmation.
- Confirmed down payment secures the booking.
- House rules email is sent when a guest email exists.
- Booking confirmation email is sent after down payment confirmation.
- Gantt-style booking board supports mouse dragging across dates and rooms.
